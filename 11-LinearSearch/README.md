# 🇬🇧 About

Linear Search is the simplest searching algorithm.

It checks each element one by one until the target value is found or the entire array has been searched.

This visualization helps users understand:

- how Linear Search works
- how searching progresses through an array
- how comparisons are performed
- why Linear Search is simple but not always efficient
- how search algorithms differ from sorting algorithms

# 🧠 How Linear Search Works

Imagine we have the following array:

```txt
12 45 7 91 33 18 64 27
```

Target value:

```txt
64
```

---

## Step 1 — Start at the Beginning

The algorithm checks the first value:

```txt
12
```

Comparison:

```txt
12 == 64 ❌
```

---

## Step 2 — Continue Searching

Next value:

```txt
45 == 64 ❌
```

Then:

```txt
7 == 64 ❌
```

And so on.

---

## Step 3 — Target Found

Eventually:

```txt
64 == 64 ✅
```

The algorithm stops because the target value has been found.

---

# 🚀 Why Use Linear Search?

Linear Search is:

✅ Simple

✅ Easy to implement

✅ Works on unsorted data

However, it becomes slower as the dataset grows because every value may need to be checked.

---

# 📊 Time Complexity

Best Case:

```txt
O(1)
```

Target is found immediately.

Worst Case:

```txt
O(n)
```

Target is at the end or does not exist.

Average Case:

```txt
O(n)
```

# 🇨🇿 Jak funguje Linear Search?

Linear Search je nejjednodušší vyhledávací algoritmus.

Postupně prochází prvky pole jeden po druhém a porovnává je s hledanou hodnotou.

Jakmile najde shodu, vyhledávání končí.

Vizualizace pomáhá pochopit:

- jak funguje Linear Search
- jak probíhá vyhledávání v poli
- jak fungují porovnání
- proč je algoritmus jednoduchý, ale nemusí být nejrychlejší
- rozdíl mezi vyhledáváním a tříděním
